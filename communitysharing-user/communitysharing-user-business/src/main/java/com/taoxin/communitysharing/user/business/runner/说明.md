# 存放项目启动时的初始化操作
## 主要方法有
1. 使用 @PostConstruct 注解
```java
@PostConstruct
public void init() {
    // todo 初始化
}
```
- 具体是在spring容器初始化bean完成后执行，适合简单的初始化逻辑，执行时机较早。
2. 使用 ApplicationRunner 接口
```java
public class ApplicationRunnerImpl implements ApplicationRunner {
    @Override
    public void run(ApplicationArguments args) throws Exception {
        // todo 启动时初始化
    }
}
```
- 具体是在spring容器启动后执行，适合需要访问命令行参数的初始化逻辑
- 原因： ApplicationRunner 启动时执行，ApplicationArguments 获取命令行参数
3. 使用CommandLineRunner接口
```java
public class CommandLineRunnerImpl implements CommandLineRunner {
    @Override
    public void run(String... args) throws Exception {
        // todo 启动时初始化
    }
}
```
- 启动时执行，适合需要访问命令行参数的初始化逻辑，执行时机在 Spring Boot 应用启动完成后。
4. 使用ApplicationReadyEvent监听器(@EventListener)
```java
@Component
public class ApplicationReadyEventListener {
    @EventListener(ApplicationReadyEvent.class)
    public void onApplicationReadyEvent(ApplicationReadyEvent event) {
        // todo 启动时初始化
    }
}
```
- 在 Spring Boot 应用完全启动并准备好服务请求时执行初始化工作。
5. 使用SmartInitializingSingleton接口
```java
@Component
public class SmartInitializingSingletonImpl implements SmartInitializingSingleton {
    @Override
    public void afterSingletonsInstantiated() {
        // todo 启动时初始化
    }
}
```
- 一种在所有单例 bean 初始化完成后执行代码的方式。
6. 使用InitializingBean接口
```java
@Component
public class InitializingBeanImpl implements InitializingBean {
    @Override
    public void afterPropertiesSet() throws Exception {
        // todo 启动时初始化
    }
}
```
- InitializingBean接口在Spring容器初始化时调用afterPropertiesSet方法，用于在bean属性设置完成后执行初始化逻辑。

# 操作
## 1. 角色与权限的关系同步到redis
### 逻辑分析：
- 先获取角色表中的部分所需字段
- 再将角色表中的id全部获取
- 根据角色id获取所有的权限表数据
- 再从权限中遍历出角色id与权限id的对应Map集合
  - 例： `1:[1,2,3,4,5]`
- 从数据库中获取所有的权限（status = 0 and isdelete = 0）
- 在以相同的方式构成权限id与实体对应的Map集合
  - 例： `1:RermissionDo{id, name, permission_key, status}`
- 从“角色id与权限id的对应Map集合”与“权限id与实体对应的Map集合”构建出角色-权限的关系集合
  1. 遍历角色表，拿到单独的角色id
  2. 通过角色id在“角色id与权限id的对应Map集合”中获取对应的权限id表
  3. 创建权限集合存储角色id对应的权限
  4. 遍历权限id表，获取单独的权限id
  5. 通过权限id在“权限id与实体对应的Map集合”中拿到有数据的对应的权限实体
  6. 将有数据的权限实体放到创建的权限集合中
  7. 最后将以角色id为key，权限集合为value的放到角色-权限的关系集合中
- 将角色-权限的关系集合以构建的redis键的前缀加上角色id作为key，角色-权限的关系集合作为value保存到redis中
